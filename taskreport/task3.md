# Task 3 — Async Logging Pipeline End-to-End Verification

## Description
Verified that the asynchronous logging pipeline works end to end: each click sent
to the engine (`POST /v1/click`) is pushed onto a Go channel, picked up by a
background goroutine, and batch-inserted into the database. The goal was to confirm
that clicks generated by the traffic generator appear as rows in the `click_logs`
table within seconds.

> Note: the project currently uses **Postgres** (table `click_logs`) rather than
> ClickHouse, which is the documented fallback in the task description.

## Setup / Build
Stack started via Docker Compose (Postgres, Redis, engine, analytics, frontend):

```
docker compose up -d
docker ps    # confirmed all containers Up
```

Engine startup logs confirmed the async logger is active:
```
[BloomFilter] Successfully loaded 12000 bad IPs into memory
Asynchronous Batch Logger started
Core Engine API Gateway started on :8080
```

Traffic generator built and run (Go):
```
go build -o generator.exe ./cmd/generator
.\generator.exe -duration 15s
```

## Verification
Row count measured before and after the generator, plus a freshness check
(rows inserted in the last minute):

```
docker exec antifraud-postgres psql -U antifraud -d analytics -c "SELECT count(*) FROM click_logs;"
docker exec antifraud-postgres psql -U antifraud -d analytics -c "SELECT count(*) FROM click_logs WHERE processed_at > now() - interval '1 minute';"
```

## Results

Generator report:
```
   ANTI-FRAUD TEST REPORT
  Traffic Mode        : NORMAL
  Duration            : 15.1s
  Total Requests Sent : 1500
  Clean Clicks (200)  : 1498
  Rate-Limit Hits(429): 0
  Blacklist Hits (403): 2
```

Database row counts:

| Metric | Value |
|--------|-------|
| Rows before | 2970 |
| Requests sent | 1500 |
| Rows after | 4470 |
| Delta | +1500 |
| Rows inserted in last minute | 1500 |

## Conclusion
All 1500 generated clicks reached the `click_logs` table within seconds. The delta
(+1500) matches the number of requests sent exactly, and the "last minute" count
confirms the rows appeared immediately — no lag, no lost rows.

Pipeline confirmed working end to end:
`POST /v1/click` -> Go channel -> background goroutine -> batch insert -> Postgres.
